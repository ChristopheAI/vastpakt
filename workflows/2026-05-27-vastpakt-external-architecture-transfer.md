# Vastpakt External Architecture Transfer

## When To Use

Use this when the user shares an external post, page, framework, or example and asks to use it for the Vastpakt homepage.

## Input Sources

- external URL or screenshot supplied by the user;
- current homepage in Browser;
- `AGENTS.md`;
- `docs/copy/2026-05-27-vastpakt-homepage-copy-classification.md`;
- `docs/experiments/homepage-experiments.tsv`;
- relevant `docs/research/` notes;
- Browser or Exa only when needed to read the external source.

## Exact Prompt

```text
[external URL or screenshot]

landing page architecture overnemen toch

okay goed doen

okay move forward in the optimal way possible
```

## Output Format

The session should output:

- what can be transferred as page discipline;
- what must not be copied;
- the Vastpakt section-job decision;
- the smallest useful homepage edit;
- updated docs;
- verification results.

Good transfers:

- proof close to claim;
- concrete before/after;
- one recognizable scene;
- a CTA placed after recognition;
- trust/friction reduction near contact.

Do not transfer:

- wording;
- visual style;
- pressure tactics;
- SaaS/AI/direct-response category language;
- broad promises.

## Review Step

Before publishing any change, ask:

1. Is this a structure/placement lesson, not copied copy?
2. Does it preserve Flemish KMO workday language?
3. Does it make the Vastpakt route clearer?
4. Did it add claims that need proof?

If the source mainly gives style or hype, document the lesson and do not edit the homepage.
