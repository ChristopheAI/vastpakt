# Loop 67 - Traject And Priority Language Grill

Date: 2026-05-27

## Question

Where does the Vastpakt homepage still drift toward a software-project, consultancy, workshop, audit, or roadmap frame?

## Exa Signals

- Onoweb frames first contact as a kennismaking followed by voorstel, timing, budget and sprints.
- LetzGo frames the start as a verkennend gesprek, first scope/cost idea, then proposal and first version.
- Mad IT frames maatwerk as an A-Z software project with idea, blueprint, sprints, testing and support.
- Analyz-it and Sitesoft use recognizable KMO pain around Excel, loose tools and standard software, but still move toward software op maat, applications, architecture, documentation and broader processes.

## Vastpakt Grill

Vastpakt should not sound like a lighter version of those pages.

The page is strongest when it stays below project language:

- one example;
- one place where someone would search again tomorrow;
- which mail, Excel, CRM, appointment or note belongs to it;
- what status, proof or next step is missing.

## Public Changes

- `Wat krijg ik terug?` became `Wat wordt duidelijk?`
- `voordat je een tool of traject kiest` became `voordat je iets verandert`
- `wat eerst klein genoeg is om vast te pakken` became `waar je eerst klein begint`
- method eyebrow `Eerste Zicht` became `Zo kijken we`
- `Eén klein stuk om vast te pakken, geen groot traject om indruk te maken` became `Eerst het stuk waar iemand morgen opnieuw zou moeten zoeken`
- `één eerste prioriteit kiezen` became `één plek kiezen om eerst helder te maken`

## Remaining Risk

The offer name `Eerste Zicht` is still unproven market language. Keep using it sparingly and explain it through ordinary workday questions.
