# Loop 93 - Examples Visual Hierarchy Grill

## Question

After adding the generated examples visual, is the examples section still proof-led or did the image become too dominant?

## Sources Used

- Current loop 92 homepage at the examples section.
- Exa pass on B2B case-study section patterns.
- Exa pass on anonymous case-study credibility.
- Local direct-response notes around specificity, credibility and visual proof.
- Prior Swipefile proof discipline already used in the Vastpakt homepage loop.

## External Signals

- Strong case-study sections keep proof close to the claim: problem, what changed, outcome, quote, screenshot, dashboard, or other concrete asset.
- Anonymous stories become weaker when names/logos are removed and no other concrete detail replaces them.
- Anonymous stories can still work when they are honest about the concrete friction and supported by artifacts, before/after details or approved metrics.
- Visuals help scanability, but they should not look like proof if they are not real screenshots or client-approved assets.

## Verdict

The generated visual is useful as a calm work-object cue, but in loop 92 it had too much vertical weight for an illustrative asset.

That makes the section slightly weaker: the visitor sees a large polished image before the actual practice cards. Since the image is not client proof, it should support the cards rather than compete with them.

## Decision

Reduce the examples visual height.

No new copy, claims, testimonials, logos, public client names, metrics or proof language were added.

## Boundary

The image remains illustrative. It must not be described or treated as a screenshot, measured result, testimonial, named case or client-approved proof.

The next real proof gain is not stronger wording. It is permission-backed screenshots, anonymized real artifacts, quotes, or publishable before/after details.
