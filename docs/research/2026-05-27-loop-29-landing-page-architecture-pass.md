# Loop 29 - Landing Page Architecture Pass

## Source

X post by The Boring Marketer:

```text
I follow this simple guide when I build a landing page.

- use playwright to get some visual inspo
- throw in a brand voice skill
- use a front end design skill
```

The attached image lists a landing page architecture:

1. Hero.
2. Success / confirmation.
3. Problem-agitate.
4. Value stack.
5. Social proof.
6. Transformation.
7. Secondary CTA.
8. Footer.

## Verdict

Partially correct for Vastpakt.

The architecture is useful as a section-job checklist. It should not be copied as sales language or visual style.

## Vastpakt Translation

- Hero: already exists.
- Success / confirmation: translated into the low-friction time-loss check plus BAVAST proof.
- Problem: already exists through the examples and problem sections.
- Value stack: already exists in `Wat je krijgt`.
- Social proof: weak but present through BAVAST.
- Transformation: do not use this frame publicly. Vastpakt should show before/after work visibility, not big transformation language.
- Secondary CTA: missing.
- Footer: missing.

## Decision

Make the smallest architecture correction:

1. Add a secondary CTA directly after the examples grid, because that is where a scroller has enough recognition to act.
2. Add a simple footer with navigation and the stable Vastpakt anchor.

## Public Copy Classification

- `Herken je één plek waar dit bij jou gebeurt?`: Vastpakt translation of the secondary-CTA job, grounded in the examples just above it.
- `Neem één mail, offerte, Excel-rij, CRM-notitie of Word-document waar je opnieuw moet zoeken, overtypen of rondvragen.`: market/workday objects and loop 21-23 check language.
- `Stuur dat voorbeeld`: light edit from the existing CTA.
- Footer line: light edit of the stable anchor.

## Remaining Risk

This pass improves page architecture, not proof depth. The site still needs more real named proof before the social-proof section can carry more weight.
