# Copywriting Selfstudy Loop

Status: living process document. Each round may amend this file; amendments are the recursive part of the loop — the loop improves the loop.

Version: v1 (2026-06-12, after rounds 1–2)

## Purpose

Improve copywriting craft for Vastpakt in a way that survives sessions. Learning persists in three places only:

```text
reviews/writing-checklist.md   -> fail-able checks (how future copy gets judged)
docs/research/*-selfstudy-*    -> round notes (why each check exists)
docs/copy/                     -> captured language (what copy is built from)
```

A round that changes none of these did not happen.

## Round protocol

1. Pick a round type:
   - `reading round`: one external source, Swipefile discipline, extract fail-able lessons.
   - `self-application round`: run previously learned tests against a real Vastpakt asset (live page, copydeck, mail, lead-log).
2. Capture before interpreting; quote exactly; classify per `rules.md`.
3. End with at least one of:
   - a new fail-able line in a `reviews/` checklist;
   - a re-testable finding on a real asset;
   - a new question routed to the werkboek-interview.
4. Never auto-edit public copy from a round. Findings become candidates; copy decisions go through the copydeck with classification.
5. Close with a meta-lesson: what did this round teach about running rounds? Amend this file if durable.

## Stop/pivot guard

If two consecutive rounds produce no checklist change and no actionable finding, switch round type. If both types are dry, stop the loop and say so — more rounds would be reading theatre.

## Lessons encoded so far

- v1 (rounds 1–2): alternate types — a reading round supplies tests, a self-application round cashes them in. Self-application yields more actionable findings per minute and can surface internal benchmarks (the mailbox-case numbers) that external sources cannot. External lessons only count once tested against our own copy.
- v1: convergence is a result, not a failure: when an external rule restates an existing Vastpakt rule (Dry's `you can't bullshit specifics` = `specific work object > abstract promise`), record the convergence and keep only the non-overlapping tools.
- v1 (round 2 correction): grep counts on HTML must separate head metadata from visible copy before a repetition verdict — round 2 counted meta tags as drift and had to be corrected. Re-verify counts against the rendered page, not the raw file.
- v1 (first shipped change): findings become shippable only when every changed line is a captured anchor or a classified light edit, and the change is logged in `docs/experiments/homepage-experiments.tsv` with sources and a follow-up verdict. That route turned round-2 finding 1 into the 2026-06-12 `proof-section-sterkste-zin` experiment.
