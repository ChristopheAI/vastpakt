# Loop 73 - Awareness-Level Alignment

## Question

After the fresh-eyes reread of Great Leads, which homepage section speaks to the wrong awareness level?

## Sources checked

- Current homepage section order.
- `2026-05-28-great-leads-reading.md`, especially the fresh-eyes awareness map.
- Current copy classification through loop 72.

## Great Leads signal

The lead and section opening should match what the visitor already knows at that point.

For Vastpakt:

- hero: problem-aware;
- examples: story/proof moving toward solution-aware;
- Eerste Zicht: solution-aware offer explanation;
- contact: direct offer/instruction.

## Verdict

The section immediately after `Eerste Zicht` still read like a new problem-aware opening:

`Waar het manuele werk begint`

and:

`Copy-paste, dubbele invoer en vergeten opvolging zijn signalen.`

That copy is not wrong, but it is in the wrong mode after the offer section. The visitor has already seen the problem and the first offer. The next section should support why visibility comes before tool choice, not restart the page.

## Change

Changed the section frame to:

`Waarom eerst zichtbaar maken?`

and:

`Copy-paste, dubbele invoer en vergeten opvolging tonen waar je moet kijken.`

## Why this is safe

- Same source-backed problem language stays present.
- The section now supports the offer logic.
- No new promise or proof claim was added.
- No internal method term was introduced.

## Boundary

Do not push `Eerste Zicht` higher as a hero-level offer name. Keep the page sequence:

problem recognition -> proof -> first action bridge -> offer explanation -> support/fit/contact.
