# Loop 32 - So-What Landing Pass

## Question

Apply the so-what test to the current Vastpakt homepage copy and move forward in the optimal way possible.

## Verdict

Better approach available.

The page already names recognizable work objects. The next improvement is to make selected lines land on the reader's consequence:

```text
What do I see after this?
What can someone follow?
What can I decide?
What can I show when there is discussion?
```

## Changes

### Hero

Before:

```text
Volgende stap niet zichtbaar.
```

After:

```text
Dan zie je waar de volgende stap verdwijnt.
```

Reason: the old line named the state. The new line tells the reader what becomes visible.

### Offer

Before:

```text
Wat daarna moet gebeuren.
```

After:

```text
Daarna zie je welk stuk werk eerst zichtbaar moet worden.
```

Reason: the offer now lands on the first useful decision.

### Method

Before:

```text
Waar komt ze binnen, waar wordt ze opnieuw overgenomen, waar blijft ze wachten, en waar verandert ze van tool?
```

After:

```text
Waar komt ze binnen, waar wordt ze opnieuw overgenomen, waar blijft ze wachten? Zo kan iemand anders het morgen ook volgen.
```

Reason: the so-what is overdraagbaarheid, not tracing for its own sake.

### Connection Cards

- `wie bezit het` -> `wie is aan zet`
- `wat is de volgende stap als het schuurt` -> `wat toon je als het schuurt`

Reason: these are more action-oriented and closer to what a KMO needs in the moment.

## Classification

- `volgende stap`: existing Vastpakt/source-backed field.
- `waar de volgende stap verdwijnt`: Vastpakt translation from the hero work-state sequence.
- `welk stuk werk eerst zichtbaar moet worden`: existing contact/offer language.
- `iemand anders het morgen ook volgen`: sourced from the recurring transfer/one-person-dependency theme.
- `wie is aan zet`: ordinary workday language, lighter than `wie bezit het`.
- `wat toon je als het schuurt`: proof/discussion language from local proof-route research.

## Remaining Risk

The hero line is now slightly more interpretive. Keep it only if browser review confirms it remains clear and not too clever.
