# Loop 47 - Deliverable Threshold Review

## Question

After loop 46, what is the next weak spot when checking Exa scan/audit pages, Swipefile friction discipline, and the rendered offer block?

## Exa Signals

- Scan/audit pages often make deliverables sound large: reports, roadmaps, recommendations, maturity, quick wins, analysis documents, and implementation paths.
- Codelines stays closer to Vastpakt when it names where info sits today: Excel, e-mail, loose systems, one process, less copying, less checking.
- Foocus and Qfor show the risk: `deliverables`, analysis documents, validation, detailed reporting, and process lists can quickly make the page feel like audit/ERP consulting.
- JeMSA is useful only as a caution: concrete work in the existing Excel flow is good, but public copy must avoid broad AI/tool claims.

## Swipefile Discipline

- At this stage of the page, the offer should reduce friction, not ask the visitor to imagine a large diagnostic package.
- The reader has already been asked for one example. Do not make the next block feel like preparation for a report.
- Replace abstract deliverable language with the actual work objects being looked at.

## Browser / Copy Grill Observation

- The `Als dit past` block still had three repeated `Dan...` lines.
- `Dan helpen echte voorbeelden...` sounded like preparation guidance, not a natural next step.
- `Een voorstel voor één eerste prioriteit` sounded like advisory/roadmap language.

## Decision

Make the offer block plainer:

- `Als dit past` -> `Daarna bekijken we`
- Replace the repeated `Dan...` bullets with ordinary objects: same situation, related mails/Excel/CRM/Teams/offertes/afspraken/notities, and how it runs today.
- `Een voorstel voor één eerste prioriteit` -> `Welk stuk werk eerst zichtbaar moet worden.`

## Classification

- `Daarna bekijken we`: light edit / sequence language.
- Related work-object list: market/workday objects already used across the homepage.
- `Welk stuk werk eerst zichtbaar moet worden`: light edit of existing contact/offer return value; removes `voorstel` and `prioriteit`.

## Remaining Risk

- `situatiekaart` and `voor/na-beeld` remain internal-ish deliverable terms, but currently have nearby explanatory fields.
