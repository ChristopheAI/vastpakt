# Charles Floate-style SEO plan for Vastpakt

Date: 2026-05-30

## Verdict

Do not copy Charles Floate's grey/black-hat edge tactics for Vastpakt.

Do borrow the operating logic:

1. establish the entity;
2. make the site look and behave like a real business;
3. build pages only where intent is clear;
4. add trust signals and local/contextual references;
5. measure leads, not vanity traffic;
6. improve based on SERP and lead evidence.

## What this means for Vastpakt

Vastpakt is not a plumber, roofer, or emergency local service.

So the correct translation is not city-page spam like:

```text
automatisering Antwerpen
automatisering Gent
automatisering Leuven
```

That would be weak and likely generic.

The stronger angle is:

```text
Vlaamse KMO's where work gets stuck between Excel, mail, WhatsApp, CRM, facturatie and afspraken.
```

## If Charles audited Vastpakt now

He would probably say the current technical base is good enough:

- site live;
- indexable;
- sitemap submitted;
- metadata and structured data present;
- contact form works;
- Resend delivery works;
- Umami funnel exists.

Then he would move away from technical perfection and toward authority plus intent.

## Priority 1: Entity stack

Create or tighten consistent Vastpakt profiles and references.

Minimum clean stack:

- LinkedIn company page;
- Christophe LinkedIn profile linked to Vastpakt;
- Google Business Profile only if the business qualifies truthfully;
- Bing Places if GBP exists;
- GitHub organization or public profile reference if relevant;
- Crunchbase only if it can be filled truthfully;
- local Belgian/Flemish business directories where NAP/category can be consistent;
- same business name, domain, email, and description everywhere.

Avoid:

- fake addresses;
- fake reviews;
- irrelevant directories;
- AI-generated profile fluff;
- press releases without a real announcement.

## Priority 2: SERP-first content map

Do not start with blog volume.

Start with 3 to 5 intent pages only if source evidence supports them:

1. `offerte-opvolging`
2. `excel-mail-opvolging`
3. `factuurroute`
4. `crm-losse-tools`
5. `werk-overdragen-bij-afwezigheid`

Each page needs:

- one concrete workday situation;
- source/market language;
- one proof object or example;
- clear first step to contact;
- internal links back to the homepage and related pages;
- FAQ only if questions are real or SERP-visible;
- one or two extractable answer chunks that can stand alone as definitions, examples, or lists.

Example chunk format:

```text
Wat is offerte-opvolging?

Offerte-opvolging is het zichtbaar houden van wie na een verstuurde offerte contact opneemt, wanneer dat gebeurt, waar het laatste antwoord staat en welke volgende stap bij het dossier hoort.
```

This is not automatically public copy. It must still be checked against source language and SERP intent before publishing.

## Priority 3: Trust and conversion

Add trust close to intent, not as generic badges.

Useful signals:

- who replies;
- what the first reply contains;
- real before/after proof objects;
- no overclaim about automation;
- delivered contact route;
- clear 24-hour reply promise;
- short examples of situations that fit and do not fit.

Avoid:

- fake case-study numbers;
- ROI claims;
- "AI automation agency" language;
- generic "save time and money" blocks.

## Priority 4: Local/contextual links

Build links that make sense for a Flemish KMO operator.

Targets:

- local entrepreneur directories;
- chamber or network pages if memberships are real;
- guest posts/interviews on KMO operations, Excel/process follow-up, or practical AI use;
- supplier/partner/client mentions when truthful;
- podcast/newsletter mentions around small business operations.

Do not use PBNs, bought junk links, fake reviews, or irrelevant guest posts.

## Priority 5: Measurement loop

Use the current funnel as the truth surface:

```text
contact_anchor_clicked -> contact_form_started -> contact_form_submitted
```

Add Search Console evidence:

- queries;
- country/device;
- page;
- impressions;
- CTR;
- position.

Add lead quality evidence:

- first 3 real leads in `docs/operations/lead-log.md`;
- situation object;
- whether the situation was usable;
- whether the reply became a conversation.

## Priority 6: Internal link graph

If Vastpakt adds intent pages, do not leave them as isolated posts.

Internal linking pattern:

- homepage links to the strongest current intent page;
- each intent page links back to the homepage contact section;
- related pages link to each other only when the work situation overlaps;
- proof/case sections link to the relevant intent page when the proof object matches;
- avoid generic footer-only linking as the main discovery path.

For example:

```text
homepage -> offerte-opvolging
offerte-opvolging -> excel-mail-opvolging
offerte-opvolging -> contact
mailbox practice case -> excel-mail-opvolging
```

## Priority 7: Agentic source-of-truth page

Because AI Overviews and agentic browsing can summarize without sending much click traffic, Vastpakt should have one crawlable source-of-truth page before scaling content.

This is not a marketing landing page.

It should help search engines and AI systems understand:

- what Vastpakt is;
- who Christophe is;
- what problems Vastpakt handles;
- what Vastpakt does not handle;
- what the first response contains;
- how to contact Vastpakt;
- which proof objects already exist;
- when the page was last updated.

Possible route:

```text
/over-vastpakt/
```

or, if kept more machine-readable:

```text
/vastpakt-info/
```

Required format:

- crawlable HTML;
- concise entity facts;
- internal links to homepage/contact/proof sections;
- JSON-LD aligned with existing `ProfessionalService`;
- no fake awards, no invented client numbers, no generic AI agency claims;
- updated date.

11. Decide the next page based on observed query/lead evidence.

## Bottom line

A Charles Floate-style version of Vastpakt would not keep polishing the homepage.

It would turn Vastpakt into a recognizable entity, then build a small set of intent pages and links around the exact work problems that produce leads.

For Vastpakt, the winning version is:

```text
entity + proof + intent page + measured contact path
```

not:

```text
more homepage polish + more technical testing
```
