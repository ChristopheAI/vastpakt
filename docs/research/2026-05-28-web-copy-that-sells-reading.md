## What this source adds

This source is specifically about web copy: first screen, scanability, editorial feel, clear objective, direct-response path, credibility, and action.

The most useful Vastpakt lesson:

The homepage should feel informative and grounded, not like an ad, but it still needs a clear direct-response path.

## Useful principles for Vastpakt

### Do not make the website look like an ad

Vastpakt implication:

- Keep the sober editorial/workday tone.
- Avoid hype, big claims, scarcity, discount, bonus or sales-page energy.
- Keep the page useful even before the visitor contacts Christophe.

### First screen matters

The first screen must stop the right visitor without requiring offer knowledge.

Vastpakt implication:

- Keep the hero problem-aware.
- Do not lead with `Eerste Zicht`.
- Keep one concrete scene above the fold.

### People scan

Web readers scan before reading deeply.

Vastpakt implication:

- Headings and subheads must carry the route.
- Cards must be skimmable.
- Long object lists can become effort if they are not clearly grouped.
- Each section needs one job.

### Inverted pyramid

Put the point first.

Vastpakt implication:

- Say the useful conclusion before background.
- Use the follow-up details to support, not delay, the main point.

### Five-question blueprint

The source's structure maps to Vastpakt:

1. What is the problem?
   Work gets unclear between mail, Excel, CRM, Word, Teams, maps and people.
2. Why hasn't it been solved?
   The issue is often between tools and people, not inside one tool.
3. What is possible?
   One situation can become visible enough to follow.
4. What is different now?
   Start with one concrete example before changing anything.
5. What should you do now?
   Send one example.

### Credibility elements

Useful credibility:

- practice cases;
- concrete work objects;
- user-provided facts;
- Christophe replies himself;
- within 24 hours;
- honest fit/no-fit.

Avoid unsupported testimonials, statistics or authority claims.

### CTA clarity

The source is strict: tell the reader exactly what to do.

Vastpakt implication:

- Keep `Stuur één voorbeeld`.
- Keep the form label concrete.
- Keep what comes back visible.
- Do not make the visitor prepare a project brief or diagnose the whole problem.

### One-directional page

The page should have one dominant route.

Vastpakt implication:

The route should remain:

problem recognition -> check -> nearby proof -> practice examples -> first-action bridge -> Eerste Zicht -> support/fit/FAQ/person/contact.

Avoid adding new side quests.

### Involvement devices

The check and form are useful involvement devices.

Vastpakt implication:

- The check should help recognition, not become a score.
- The form should ask for one example, not a broad diagnosis.

## What Vastpakt should not import

- Emotional exaggeration.
- Hypnotic/embedded command tricks.
- Scarcity, bonuses, discounts or deadline pressure.
- Opt-in/list-building mechanics unless there is a real newsletter/content path.
- Sales-page formulas that make the site feel like an ad.
- Webcopy cliches, corporate speak, or tentative adjectives.

## Practical grill for next loops

Ask:

1. Can the visitor understand this section by scanning only the heading and first line?
2. Does the page feel informative before it feels promotional?
3. Is the next action obvious?
4. Does this section support the one route, or create a side quest?
5. Is the first screen problem-aware and concrete?
6. Are we using enough proof without making the page heavy?
7. Is any sentence written like an ad instead of a workday explanation?

## Current implication for Vastpakt

The homepage is directionally aligned:

- hero is problem-aware;
- check creates involvement;
- examples create credibility;
- Eerste Zicht explains the small offer;
- contact gives a clear action.

The next strict risk is scanability:

Some sections carry long object lists or dense paragraphs. The next useful pass should check whether a skimming visitor can still follow the route without reading every detail.
