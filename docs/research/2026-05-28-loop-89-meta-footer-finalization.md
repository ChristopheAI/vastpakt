# Loop 89 - Meta And Footer Finalization

## Source pass

- Local homepage after loop 88.
- Exa competitor pass across Belgian/custom-software and automation pages.
- Direct-response notes: one reader, one problem, one action; reduce doubt at the edge of the action.

## Finding

The visible route is now built around one submitted workday example:

- hero CTA: `Stuur één voorbeeld`;
- offer/contact: `Stuur één voorbeeld uit je werkdag`;
- form objects: mail, offerte, Excel-rij, CRM-notitie, Word-document or afspraak;
- return promise: within 24 hours, fit, where the example hangs, and which question must become visible first.

The meta description still described the page as a broad visibility service around manueel werk, opvolging, bewijs and factuurmomenten. That is not wrong, but it is less aligned with the current direct-response route.

The footer also still used `één stuk werk`, which is less precise than the current page's repeated work object: `één voorbeeld uit je werkdag`.

## Decision

Align the public page edges with the current one-example route:

- Replace the meta description with a one-example snippet tied to the contact objects and 24-hour response promise.
- Replace the footer line with a smaller one-example line.

## Boundary

No new offer, proof, outcome, price, social-proof, urgency, project-intake, preparation, automation or savings claim was added.
