# Loop 91 - Anonymous Proof Boundary

## Question

After loop 90, should the homepage add more proof language around the anonymized practice cases?

## Sources used

- Current loop 90 homepage examples grid.
- Local Swipefile proof-system spec.
- Local practice-case notes for the four examples.
- Exa pass on anonymous case-study credibility:
  - Proofmap: anonymous case studies need richer data, context and before/after comparison when the client name is absent.
  - Orange Marketing: anonymous cases should compensate with quantified results and visual proof where possible.
  - Equinet: anonymous cases need approval, rules of anonymity, context, tangible results and enough detail to feel real.
  - Velocity Partners: anonymous cases must not become a public reference with the name removed; the value is honest friction and useful specificity.

## Verdict

Do not add stronger public proof language yet.

The current cards already contain the safe parts:

- work context;
- where the work became unclear;
- what became visible;
- concrete work objects;
- bounded change or follow-up.

The cards do not yet have enough approved public evidence for:

- client logos;
- testimonials;
- public client names;
- stronger result metrics;
- "many companies" or popularity claims;
- screenshots that might reveal private details.

## Strongest Next Proof Work

The next proof improvement is not copywriting. It is collecting publishable proof assets:

1. For each case, decide if it may be named, anonymized, or kept internal.
2. For anonymized cases, agree which context may be public.
3. Capture one safe visual or output proof where possible:
   - blurred screenshot;
   - empty/demo workbook sheet;
   - plugin package/validation screenshot;
   - Word-template before/after crop;
   - inventory dashboard with fake data.
4. Ask for one approved sentence from the user/client only when it exists.

## Boundary

Until that exists, keep the homepage at the current proof level. Do not compensate for missing public proof with stronger wording.
