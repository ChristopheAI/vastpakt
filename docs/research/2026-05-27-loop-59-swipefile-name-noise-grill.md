# Loop 59 - Swipefile Name Noise Grill

## Question

Where does the homepage still create doubt after the person/contact route was tightened?

## Swipefile grill used

- Every section must reduce doubt.
- One reader, one problem, one action.
- Say less when the object explains.
- Make Eerste Zicht the sensible middle option.

## Verdict

The weakest spot was not the photo or layout. It was name noise between `Eerste Zicht` and `kort eerste zicht`.

Internally the distinction was clear:

- `Eerste Zicht` = offer/scan layer.
- `kort eerste zicht` = first response after one submitted example.

But public copy should not ask the reader to hold that distinction.

## Change

- Changed person card heading from `Kort eerste zicht` to `Binnen 24 uur`.
- Replaced the person response sentence with a direct description of what comes back.
- Replaced the contact return line so it no longer uses `kort eerste zicht`.

## Boundary

- `Eerste Zicht` remains an offer name, not proven market language.
- The first response is now ordinary language, not a second named object.
- No new report, audit, roadmap, score, ROI, savings, implementation, or automation claim was added.
