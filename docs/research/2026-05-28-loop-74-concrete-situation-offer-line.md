# Loop 74 - Concrete Situation Offer Line

## Question

How should the Cashvertising lesson be used without publishing internal strategy?

## Sources checked

- `2026-05-28-cashvertising-reading.md`
- Current `Eerste Zicht` offer section.
- User-approved internal rule: one concrete work situation, enough proof, easy next action.
- Copy classification through loop 73.

## Verdict

The internal rule is strong, but it should not be published literally:

`Laat de bezoeker één concrete werksituatie voor zich zien, geef genoeg bewijs om het te geloven, en maak de volgende actie makkelijk.`

That sentence describes our copy job, not the visitor's workday.

## Change

Changed the `Eerste Zicht` lede from:

`We kijken waar info binnenkomt. Waar ze blijft zitten. Wat wacht. Daarna zie je welk stuk werk eerst zichtbaar moet worden.`

to:

`Toon één voorbeeld uit je werkdag. Dan kijken we wat daar loskomt en welke vraag eerst zichtbaar moet worden.`

## Why this is safe

- It keeps the action small.
- It names the concrete starting point.
- It stays tied to one workday example.
- It does not add a new proof claim.
- It does not promise a report, audit, roadmap, score, implementation, or automation result.

## Boundary

Keep this as offer clarity, not as a new hero. The hero should remain problem-aware; this offer line belongs after proof and the scroller CTA.
