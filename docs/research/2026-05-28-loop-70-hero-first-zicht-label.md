# Loop 70 - Hero First-Zicht Label

## Question

After loop 69, does the hero still depend on unproven Vastpakt language too early?

## Sources checked

- Rendered loop 69 homepage.
- Local copy classification around `Eerste Zicht`.
- Exa scan of Belgian/KMO automation, maatwerksoftware and first-step pages.

## Source signal

Adjacent pages tend to name the first step with broad category terms:

- audit;
- scan;
- review;
- analyse;
- gesprek;
- procesanalyse;
- haalbaarheidscheck;
- roadmap.

Vastpakt should not copy those category words, because the homepage deliberately stays smaller: one concrete example before anything larger changes.

## Verdict

The hero card still had a small public label: `Na eerste zicht`.

That label asks the reader to understand the offer name before the offer section has explained it. The first viewport should be readable from the work object alone: one sent offer, one missing follow-up, and what becomes visible afterward.

## Change

Replace:

`Na eerste zicht`

with:

`Daarna`

## Boundary

Keep `Eerste Zicht` as the offer section label for now. Do not spread it into the first proof object unless market or reader evidence later supports the term.
